# Thurr Solutions Operating System

Updated: 2026-05-06

## Core Rule

Notion is the source of truth. Linear is the execution board. GitHub and local repos store code, memory, and version history. n8n moves data. Google Drive stores client-facing documents and assets. Codex uses all of it to plan, critique, build, and verify.

## Operating Map

| Area | Tool | Purpose |
| --- | --- | --- |
| Client fulfillment | Notion | Client profiles, battle plans, status, scope, assets needed, compliance notes |
| Execution | Linear | Tickets, due dates, blockers, templates, build tasks |
| Code and memory | GitHub / local repos | Website, Thurnos memory, n8n backups, docs, version history |
| Automation | n8n | Intake routing, form handling, client updates, workflow orchestration |
| Website leads | Supabase + n8n | Lead audit submissions, intake tracking, follow-up routing |
| Sales assets | Google Drive / Slides / Notion | Proposals, decks, client-facing PDFs, presentation scripts |
| Content engine | NotebookLM / Runway / Codex | Scripts, video briefs, founder voiceover planning, social repurposing |
| Reminders | Google Calendar / Codex reminders | Follow-ups, renewals, review dates, launch windows |

## What Goes Where

- Strategy, decisions, client context, and battle plans go in Notion.
- Anything that requires action goes in Linear.
- Anything that is code, prompts, memory, or repeatable documentation should also live in a repo.
- Anything automated should be documented in Notion and backed up from n8n.
- Anything client-facing should have a clean link and no internal risk notes.
