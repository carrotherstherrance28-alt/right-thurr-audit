# Pilot Shot Plan — Website Landlord / Rank-and-Rent Lead Flow

Use this as the second pilot for the Branded AI Video Production Kit.

Source context: Granola note from May 5, 2026 titled "Website landlord model overview."

Scope: educational / sales explainer for Thurr-owned content. This should explain the business model without guaranteeing rankings, leads, revenue, client retention, or profit. Keep examples generic and privacy-safe.

Right Thurr interpretation: this can become a curated Right Thurr use-case page, where the model is "right there" for Thurr to copy, adapt, and brand. The video should therefore feel like a clear template for a local lead asset, not a promise of passive income.

Related asset draft (visual direction + 9:16 script + prompts + page outline):

- `docs/project/video-production-kit/Website-Landlord-Use-Case-Asset.md`

## 1) Creative Direction (Translator)

### Brand Tone

Pick 2–3:

- practical
- high-trust/professional
- local/community

Tone notes:

- What it should feel like in 1 second: a clear local lead-flow asset, not a guru-style money pitch.
- What it should *not* feel like: guaranteed passive income, flashy cash graphics, fake search rankings, or manipulative scarcity.

### Visual Anchors (Non-Negotiables)

- Anchor 1: A niche + city website asset, such as "Junk Removal Austin" or "Roof Repair Tulsa."
- Anchor 2: Search visibility turning into tracked calls or form leads.
- Anchor 3: Lead flow being rented to a local operator for a flat monthly fee.

### Pacing Pattern

- fast hook → model explanation → proof logic → CTA

### Camera / Motion Logic

Rules:

- One camera move per shot.
- One clear subject per shot.
- Keep search pages, phone numbers, and dashboards generic.
- Use placeholder brands and numbers only.
- Text overlays should be added in Remotion when accuracy matters.

Allowed camera moves:

- locked camera
- slow push / dolly in
- gentle pan (left/right)
- slow pull back
- close-up insert push

### Quality Boundaries (Must Avoid)

- promising page-one rankings
- showing real client names, phone numbers, or private lead data
- fake Google UI that implies an actual ranking claim
- cash stacks, yachts, luxury bait, or course-seller imagery
- overcomplicated SEO charts that distract from the simple model

## 2) Shot Plan

Target: 35–50 seconds total. Keep each shot ~4–6 seconds.

### Shot 1 — "Own the lead path"

- Goal (what it proves / what it should make them feel): The asset is the website and local lead path, not the service business itself.
- Input asset (screenshot/photo/logo/graphic): Generic local-service landing page mockup: "Junk Removal Austin" with a clear call button.
- Visual anchor (what must stay stable): The website hero + call CTA.
- Camera motion (one move): slow push in.
- Scene motion (what moves in the scene, if anything): CTA button pulse once.
- Overlay text (short, mobile readable): "You can own the lead path."
- Tool recommendation: Remotion or Runway.
- Retry notes (what to adjust if it drifts): If Runway warps the site text, generate the background only and add UI/text in Remotion.

### Shot 2 — "Pick a niche and city"

- Goal (what it proves / what it should make them feel): The model starts with specific, lower-competition local markets.
- Input asset (screenshot/photo/logo/graphic): Simple map/grid of local service niches and cities.
- Visual anchor (what must stay stable): One selected tile: "Junk Removal + Austin."
- Camera motion (one move): locked camera.
- Scene motion (what moves in the scene, if anything): Selected tile highlights orange.
- Overlay text (short, mobile readable): "Niche + city first."
- Tool recommendation: Remotion.
- Retry notes (what to adjust if it drifts): Keep this crisp and graphic; no need for AI generation.

### Shot 3 — "Build the site"

- Goal (what it proves / what it should make them feel): The site is built around conversion: trust, service pages, phone, form.
- Input asset (screenshot/photo/logo/graphic): Website sections: service proof, reviews, call button, form.
- Visual anchor (what must stay stable): The call button and form.
- Camera motion (one move): gentle pan downward.
- Scene motion (what moves in the scene, if anything): Form field highlights; review card slides in.
- Overlay text (short, mobile readable): "Build for calls and forms."
- Tool recommendation: Runway or Remotion.
- Retry notes (what to adjust if it drifts): Use fewer UI blocks if the model creates clutter.

### Shot 4 — "Search visibility"

- Goal (what it proves / what it should make them feel): Ranking/visibility creates attention, but claims must stay careful.
- Input asset (screenshot/photo/logo/graphic): Abstract search-result style cards, not a real Google result.
- Visual anchor (what must stay stable): A generic local search card rising into view.
- Camera motion (one move): slow pull back.
- Scene motion (what moves in the scene, if anything): A visibility meter moves from low to medium/high without exact promises.
- Overlay text (short, mobile readable): "Earn local visibility."
- Tool recommendation: Remotion.
- Retry notes (what to adjust if it drifts): Avoid exact rank labels like "#1"; use "visibility" language.

### Shot 5 — "Track the calls"

- Goal (what it proves / what it should make them feel): Lead value is measurable through tracking phone numbers and forms.
- Input asset (screenshot/photo/logo/graphic): Privacy-safe call tracking dashboard with placeholder lead cards.
- Visual anchor (what must stay stable): A tracked call card with no real phone number.
- Camera motion (one move): close-up insert push.
- Scene motion (what moves in the scene, if anything): One "new call" card appears.
- Overlay text (short, mobile readable): "Track every inquiry."
- Tool recommendation: Runway or Remotion.
- Retry notes (what to adjust if it drifts): Use fake data only; add text overlays in Remotion.

### Shot 6 — "Rent the lead flow"

- Goal (what it proves / what it should make them feel): The local operator pays for access to the incoming lead flow.
- Input asset (screenshot/photo/logo/graphic): Lead cards routing from website to a local business inbox/phone.
- Visual anchor (what must stay stable): Arrow from lead source to local operator.
- Camera motion (one move): gentle pan left-to-right.
- Scene motion (what moves in the scene, if anything): One lead card travels across the path.
- Overlay text (short, mobile readable): "Rent the flow to an operator."
- Tool recommendation: Remotion.
- Retry notes (what to adjust if it drifts): Keep the visual as a simple routing diagram.

### Shot 7 — "Flat monthly economics"

- Goal (what it proves / what it should make them feel): The model is simple to explain: low overhead, monthly fee, value tied to lead flow.
- Input asset (screenshot/photo/logo/graphic): Clean economics card with ranges, framed as examples.
- Visual anchor (what must stay stable): "Low overhead" and "monthly flat fee" labels.
- Camera motion (one move): locked camera.
- Scene motion (what moves in the scene, if anything): Two numbers fade in as example ranges.
- Overlay text (short, mobile readable): "Simple monthly pricing."
- Tool recommendation: Remotion.
- Retry notes (what to adjust if it drifts): Use "example" and "depends on market" language; do not imply guaranteed revenue.

### Shot 8 — "Trial / handoff"

- Goal (what it proves / what it should make them feel): Local operators can test lead quality before committing.
- Input asset (screenshot/photo/logo/graphic): One-week trial card + call forwarding toggle.
- Visual anchor (what must stay stable): Trial card and forwarding switch.
- Camera motion (one move): slow push in.
- Scene motion (what moves in the scene, if anything): Forwarding toggle turns on.
- Overlay text (short, mobile readable): "Let the calls prove value."
- Tool recommendation: Runway or Remotion.
- Retry notes (what to adjust if it drifts): Keep this operational, not hype-heavy.

### Shot 9 — "Thurr angle"

- Goal (what it proves / what it should make them feel): This connects the model to Right Thurr's copyable use-case library and Thurr Solutions' implementation strengths.
- Input asset (screenshot/photo/logo/graphic): Right Thurr use-case page mockup beside a Thurr Solutions system map: website → tracking → follow-up → reporting.
- Visual anchor (what must stay stable): Four-step system map.
- Camera motion (one move): slow push in.
- Scene motion (what moves in the scene, if anything): "Copy use case" badge appears; each system step lights up once.
- Overlay text (short, mobile readable): "The play is right there. The system still has to work."
- Tool recommendation: Remotion.
- Retry notes (what to adjust if it drifts): Keep the line readable and avoid tiny dashboard details.

### Shot 10 — "CTA / next action"

- Goal (what it proves / what it should make them feel): Clear next step for content viewers or prospective clients.
- Input asset (screenshot/photo/logo/graphic): Thurr Solutions final card.
- Visual anchor (what must stay stable): CTA.
- Camera motion (one move): locked camera.
- Scene motion (what moves in the scene, if anything): Subtle orange underline sweep.
- Overlay text (short, mobile readable): "Want this mapped for your niche? Start with the audit."
- Tool recommendation: Remotion.
- Retry notes (what to adjust if it drifts): Keep it as a crisp brand card, not generated text.

## 3) Assembly Notes (If Using Remotion)

- Intro/outro brand frames: Open with Shot 1 or Shot 2; close with Shot 10.
- Safe text zones (mobile): Keep overlays centered and outside the bottom 20%.
- Caption rules (optional): Use short plain-English captions; avoid "guaranteed" language.
- Export variants: 9:16 for short-form education; 16:9 if embedding in a sales page or deck.

## Compliance / Positioning Notes

- Say "can," "model," "example," and "depends on market" instead of promising outcomes.
- Avoid claims that a site will rank or a client will pay a certain amount.
- Keep any economics framed as examples from the source note, not guarantees.
- If this becomes a client offer, pair it with the Lead Flow Audit so the build still starts with diagnosis.
